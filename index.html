01

Board Anatomy

Reset Button

Restarts the running program from the very beginning. Doesn't erase code — just re-runs setup().

USB Port

Uploads code from your computer and powers the board at the same time.

Poly Fuse

A self-healing safety guard. Trips if too much current flows, recovers automatically once the fault clears.

ATmega16U2 USB↔Serial

Translator chip between your computer and the main microcontroller — carries code uploads and Serial Monitor traffic over USB.

Voltage Regulator

Locks the supply at a steady 5V, protecting the board from an overvoltage external supply.

DC Jack

Round socket for external power (adapter / battery) when USB isn't connected.

ATmega328P main chip

The brain. Stores your code, runs the program, and drives every pin.

Crystal Oscillator 16 MHz

The board's clock — ticks 16 million times a second so timing (like delay(1000) meaning a real second) stays accurate.

Capacitors

Tiny energy tanks. Keep the supply voltage stable and smooth out noise when a sensor or motor suddenly draws more current.

Pin Map

Digital Pins 0–13

14 pins, each switchable between INPUT (reads HIGH/LOW) and OUTPUT (drives HIGH/LOW). HIGH = 5V · LOW = 0V.

0 (RX)1 (TX)2 ~34~5 ~678 ~9~10~11 1213

PWM Pins ~ symbol

3, 5, 6, 9, 10, 11 — can output "partial ON" signals for LED dimming or motor speed control via analogWrite().

Analog Pins A0–A5

Read voltages 0–5V and convert to a number 0–1023 via the ADC. Used with sensors that give changing values (light, temperature, sound).

A0A1A2 A3A4A5

TX / RX — Serial Communication

TX = Transmit (send) · RX = Receive (get). Pin 1 = TX, Pin 0 = RX.

  • Upload: computer TX → Arduino RX
  • Serial Monitor: Arduino TX → computer RX
  • Onboard TX/RX LEDs blink to show data flowing

03

Breadboard Basics

Power Rails (side lines)

Long vertical columns marked + (red) and – (blue/black) running the length of the board. Every hole in a column is internally connected — used to distribute 5V and GND to the whole circuit.

Terminal Strips (middle area)

Split into two halves by a center gap. Each row of 5 holes is connected vertically — this is where resistors, LEDs, and ICs get plugged in.

RGB LED

One LED package, three color channels (Red/Green/Blue) — mixed brightness produces any color, like a screen pixel.

Common Cathode (CC)

All negative pins tied to GND. Each color pin driven HIGH to turn it on. Most common in Arduino kits.

Common Anode (CA)

All positive pins tied to +5V. Each color pin driven LOW to turn it on.

Pinout (4 pins)

Pin 1 → Red Pin 2 → Common (GND/+5V) Pin 3 → Green Pin 4 → Blue

Sketch: cycling blink

int redPin = 9;
int greenPin = 10;
int bluePin = 11;

void setup() {
  pinMode(redPin, OUTPUT);
  pinMode(greenPin, OUTPUT);
  pinMode(bluePin, OUTPUT);
}

void loop() {
  // Red blink
  digitalWrite(redPin, HIGH);
  delay(1000);
  digitalWrite(redPin, LOW);
  delay(500);

  // Green blink
  digitalWrite(greenPin, HIGH);
  delay(1000);
  digitalWrite(greenPin, LOW);
  delay(500);

  // Blue blink
  digitalWrite(bluePin, HIGH);
  delay(1000);
  digitalWrite(bluePin, LOW);
  delay(500);
}
06

Push Button

A button connected with no resistor leaves the pin "floating" — it reads random noise when not pressed. A pull resistor fixes the default state.

Pull-Down

Resistor to GND. Default = LOW. Pressed → connects to 5V → reads HIGH.

Pull-Up

Resistor to 5V. Default = HIGH. Pressed → connects to GND → reads LOW.

08

9V Battery

+ terminal is the energy outlet, – terminal is the return path — current loops from + through the components and back to –.

  • Used in remotes, toys, smoke detectors, and Arduino (via Vin or barrel jack)
  • Never connect + directly to – — that's a short circuit and will damage the battery
  • Always pair an LED with a 220–330Ω resistor
  • Supplies more current than Arduino pins can — good for motors or several LEDs at once
09

Potentiometer

A resistor whose value changes as you turn a knob, via a sliding contact called the wiper.

Side pin → 5V Side pin → GND Middle (wiper) → A0

Wiper near 5V side → high output · near GND side → low output · in between → a voltage from 0–5V.

10

Ultrasonic Sensor (HC-SR04)

Measures distance by timing a sound pulse's round trip — same principle bats and dolphins use.

Distance = (Speed of sound × Time) ÷ 2 — divided by 2 because the wave travels out and back.

PinFunction
VCC+5V
TrigSends the pulse OUTPUT
EchoReceives the reflection INPUT
GNDGround
int trigPin = 9;
int echoPin = 10;
long duration;
int distance;

void setup() {
  pinMode(trigPin, OUTPUT);
  pinMode(echoPin, INPUT);
  Serial.begin(9600);
}

void loop() {
  digitalWrite(trigPin, LOW);
  delayMicroseconds(2);

  digitalWrite(trigPin, HIGH);
  delayMicroseconds(10);
  digitalWrite(trigPin, LOW);

  duration = pulseIn(echoPin, HIGH);
  distance = duration * 0.034 / 2;

  Serial.print("Distance: ");
  Serial.print(distance);
  Serial.println(" cm");

  delay(500);
}

Micro Servo Motor

Unlike a DC motor that spins continuously, a servo moves to a specific angle (0°–180°) and holds it. It's an output device, driven by PWM.

#include <Servo.h>

Servo myServo;

void setup() {
  myServo.attach(9);
}

void loop() {
  myServo.write(90);   // 0°=min · 90°=center · 180°=max
}

Real-world uses

Robot arms · automatic doors · smart dustbins · parking gates · pick-and-place robots.

13

pinMode()

Core functions — the commands that actually move data on and off the board.

pinMode(pin, mode)

Tells a pin whether it should listen (INPUT) or speak (OUTPUT). Always called once inside setup() before you use the pin for anything else.

mode: INPUT · OUTPUT · INPUT_PULLUP (enables the chip's internal pull-up resistor, so you can skip an external one)

void setup() {
  pinMode(13, OUTPUT);     // pin 13 drives an LED
  pinMode(2, INPUT_PULLUP); // pin 2 reads a button
}
14

digitalWrite()

digitalWrite(pin, value)

Sets an OUTPUT pin fully ON or fully OFF. No in-between — this is a light switch, not a dimmer.

value: HIGH (5V) · LOW (0V)

void loop() {
  digitalWrite(13, HIGH); // LED on
  delay(1000);
  digitalWrite(13, LOW);  // LED off
  delay(1000);
}
15

digitalRead()

digitalRead(pin)

Checks an INPUT pin and reports back HIGH or LOW. The classic use is reading a button or a digital sensor's OUT pin.

int state = digitalRead(2);
if (state == HIGH) {
  Serial.println("Button pressed");
}
16

analogWrite()

analogWrite(pin, value)

Fakes an analog output using PWM — rapidly switching the pin on and off so fast it looks like a dimmed voltage. Only works on PWM pins, marked with a ~: 3, 5, 6, 9, 10, 11.

value: 0 (always off) to 255 (always on)

void loop() {
  analogWrite(9, 128); // LED at ~50% brightness
}
17

analogRead()

analogRead(pin)

Reads a changing voltage on an analog pin (A0–A5) and converts it to a number using the onboard ADC. Used for sensors that output a range rather than just on/off.

returns: 0 (0V) to 1023 (5V)

int light = analogRead(A0);
Serial.println(light);
18

tone()

tone(pin, frequency, duration)

Makes a piezo buzzer sing a steady note by vibrating the pin at a chosen frequency. duration (ms) is optional — leave it out and the tone plays until noTone() stops it.

tone(8, 440, 500); // A4 (440Hz) for 500ms
19

noTone()

noTone(pin)

Stops whatever tone() started on that pin. Needed if you called tone() without a duration, or want to cut a note short.

tone(8, 262);  // note starts, plays forever
delay(300);
noTone(8);     // silence
20

delay()

delay(ms)

Pauses the whole program for the given number of milliseconds. Simple, but the board can't do anything else — read a button, check a sensor — while it waits.

delay(1000); // pause for 1 second
22

map()

map(value, fromLow, fromHigh, toLow, toHigh)

Rescales a number from one range into another — like a translator between scales. Commonly used to turn a raw analogRead() (0–1023) into something useful, like a servo angle (0–180) or PWM value (0–255).

Example: converting 0–100°C into 32–212°F is exactly what map() does in one line.

int raw = analogRead(A0);                  // 0-1023
int angle = map(raw, 0, 1023, 0, 180); // 0-180
23

pulseIn()

pulseIn(pin, value)

Measures how long (in microseconds) a pin stays HIGH or LOW. Its signature job is reading the Echo pin of an ultrasonic distance sensor.

long duration = pulseIn(echoPin, HIGH);
int distanceCm = duration * 0.034 / 2;

Serial.begin() / print() / println()

Serial.begin(baudRate)

Opens a communication channel between the Arduino and your computer over USB (using the TX/RX pins). begin() sets the speed once in setup(); print()/println() send text to the Serial Monitor, the latter adding a line break.

void setup() {
  Serial.begin(9600);
}
void loop() {
